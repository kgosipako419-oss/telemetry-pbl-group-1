## Project Overview

This repository contains the semester-long Problem Based Learning laboratory project for TELE 523. The project designs and simulates a complete end-to-end telemetry and remote monitoring system using Python, applied to the domain of **industrial machine condition monitoring**.

The system ingests real sensor data from the Fischertechnik Smart Factory IoT dataset, processes it through analog and digital modulation and demodulation stages, applies digital telemetry techniques, extracts machine condition features, and displays results on a Streamlit monitoring dashboard with threshold-based alerting.

---

## Application Domain

**Industrial Machine Condition Monitoring**

Target signals: IMU accelerometer data, actuator state signals, UWB position data
Target tasks: Event classification, anomaly detection

---

## Dataset

**Fischertechnik Smart Factory IoT Dataset**

- Source: [https://github.com/iml130/fischertechnik-dataset](https://github.com/iml130/fischertechnik-dataset)
- Description: Multi-sensor recordings from a physical factory testbed including IMU, UWB position, and actuator state data collected under real operating conditions.
- Raw data is stored unmodified in `data/raw/`. See `data/README.md` for sampling strategy, segment justification, and full citation.

## Setup Instructions

### 1. Clone the repository

```bash
git clone https://github.com/<your-username>/telemetry-pbl-group-01.git
cd telemetry-pbl-group-01
```

### 2. Create and activate a Python environment

```bash
conda create -n tele523 python=3.10
conda activate tele523
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Add the dataset

Download the Fischertechnik dataset and place the raw files in:
```
data/raw/
```
See `data/README.md` for the exact files required and the download link.

---

## Running the Pipeline

Run each stage in order from the repository root:

```bash
# Step 1 — Preprocess the raw dataset
python src/preprocessing.py

# Step 2 — Run modulation and channel simulation
python src/modulation.py

# Step 3 — Run demodulation and compute BER
python src/demodulation.py

# Step 4 — Run digital telemetry (quantization, PCM, line coding)
python src/digital_telemetry.py

# Step 5 — Extract features
python src/features.py
```

All outputs are saved to `data/processed/`, `results/`, and `figures/` automatically.

---

## Launching the Dashboard

```bash
streamlit run src/dashboard.py
```

The dashboard reads from files in `data/processed/` and `results/`. The full pipeline must be run at least once before launching the dashboard.

---

## Reproducing All Figures

All figures used in the report are generated by the pipeline scripts above and saved to `figures/`. To regenerate all figures from scratch:

```bash
conda activate tele523
pip install -r requirements.txt
python src/preprocessing.py
python src/modulation.py
python src/demodulation.py
python src/digital_telemetry.py
python src/features.py
```

## Git Workflow

All contributors must follow these rules:

- **Never push directly to `main`**
- Create a branch per task: e.g. `feature/preprocessing`, `feature/modulation`
- Write clear commit messages: e.g. `"Add FIR filter to preprocessing module"`
- Open a pull request and request review before merging into `main`
- Never modify files in `data/raw/`
