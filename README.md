## Overview

This repository contains the semester-long Problem Based Learning (PBL) laboratory project
for TELE 523 — Telemetry and Remote Control at BIUST. The project designs, simulates, and
validates a complete end-to-end telemetry and remote monitoring system using Python.

The system is applied to the domain of **industrial machine condition monitoring**, ingesting
real sensor data from the Fischertechnik Smart Factory IoT dataset and passing it through a
fully implemented telemetry pipeline. The final output is a Streamlit monitoring dashboard
with real-time feature visualisation and threshold-based alerting.

The project demonstrates competence across four core areas defined by the TELE 523 course
objectives:

- Telemetry architectures across analog, digital, biomedical, and industrial contexts
- Analog and digital modulation and demodulation with performance evaluation
- Digital telemetry with quantization, PCM encoding, and bitstream integrity verification
- Python-based monitoring with signal visualisation, feature extraction, and alerting

## Setup Instructions

### Prerequisites

- Python 3.10 or later
- Git
- VS Code (recommended IDE)

### 1. Clone the repository

```bash
git clone https://github.com/<your-username>/telemetry_pbl_group1.git
cd telemetry_pbl_group1
```

### 2. Create and activate a virtual environment

**Windows:**
```bash
python -m venv venv
venv\Scripts\activate
```

**Mac/Linux:**
```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Add the dataset

Download the Fischertechnik dataset from the source link in `data/README.MD/`.

## Running the Pipeline

### Run the full pipeline in one command

```bash
python src/system_architect/pipeline_controller.py
```

### Run each stage individually

```bash
# Stage 1 — Preprocessing
python src/signal_processing_lead/preprocessing.py

# Stage 2 — Modulation
python src/modulation_lead/am_modulation.py
python src/modulation_lead/fm_modulation.py
python src/modulation_lead/ask_modulation.py
python src/modulation_lead/fsk_modulation.py
python src/modulation_lead/psk_modulation.py

# Stage 3 — Channel and demodulation
python src/modulation_lead/channel_noise.py

# Stage 4 — Digital telemetry
python src/digital_telemetry_lead/quantization.py
python src/digital_telemetry_lead/pcm_encoding.py
python src/digital_telemetry_lead/line_coding.py
python src/digital_telemetry_lead/bit_integrity_check.py

# Stage 5 — Feature extraction
python src/monitoring_lead/feature_extraction.py
```

All outputs are saved automatically to `data/processed/`, `results/figures/`,
and `results/logs/`.

## Launching the Dashboard

The dashboard must be launched after the pipeline has been run at least once.

```bash
streamlit run src/monitoring_lead/dashboard_streamlit.py
```

The dashboard reads from processed files only and does not re-run the pipeline.
It displays signal plots, modulation outputs, BER curves, SQNR results, feature
trends, and a colour-coded alert log with CSV export.

## Git Workflow

```
feature/branch  →  dev  →  main
```

| Rule | Description |
|---|---|
| Never push to `main` directly | All changes go through a pull request |
| One branch per task | e.g. `feature/preprocessing`, `feature/modulation` |
| Clear commit messages | e.g. `"Add FIR filter to preprocessing module"` |
| Merge only after review | Pull request must be reviewed before merging to `dev` |
| Never modify `data/raw/` | Raw dataset files are read-only |

## Reproducibility

All results in the final report are fully reproducible from this repository:

1. All figures are generated by scripts in `src/` and saved to `results/figures/`
2. Results regenerate from a clean environment using only `requirements.txt`
3. Raw data is held unmodified in `data/raw/`
4. All pipeline parameters are saved to JSON metadata files in `data/processed/`

To regenerate all results from scratch:

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
python src/system_architect/pipeline_controller.py
```
